# Remarque sur l'exercice

## Général

Par manque de temps, j'ai fait ces choix de développement :

- Pas de test dans les composants, il est impératif d'en avoir sur app
- Certaines fonctionnalités ne sont pas abouties mais le fonctionnement général attendu est bien là de bout en bout.
- Le temps de review a été limité

Ce que j'ai pu faire :

- Assurer le service de bout en bout
- Développer tous les écrans demandés
- Ajuster les APIs pour une simulation de navigation immersive
- Les erreurs, loading et contenu vide ont bien été implémentés

## Page Cataloque

- Dans les maquettes, on voit : "Row per page", je n'ai pas personnalisé le composant.
- Je n'ai donc pas ajouté le nombre de produits total puisqu'il est indiqué dans le composant de pagination.
- Les icônes de pagination ne sont pas centrées
- La fonctionnalité de sélection et d'ajout multiple n'a pas été développée

## Ajustements perfectibles

- style de le snackbar
- on peut faire mieux sur le responsive
- les animations n'ont pas été travaillées
- les URLs devraient être mises à jour de façon plus précise

## Utilisation de Copilot

Les instructions de git copilot pourraient être perfectibles.
Je n'ai pas eu le temps de l'ajuster au cours du développement.
Je l'ai utilisé principalement pour générer des squelettes. Il y a peut-être des façons plus avancées de son utilisation en utilisant un MCP Figma.
